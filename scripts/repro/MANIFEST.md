# Civitas 6.7B Reproducibility Manifest

## Run Identity

This bundle defines the reproducibility manifest contract for the Civitas 6.7B
governed self-improvement paper-eval smoke pipeline.

| Field | Value |
| --- | --- |
| Run id | Generated by `RUN_ALL.sh` under `repro/runs/<run_id>/MANIFEST.md` |
| Timestamp | Generated from `CIVITAS_67B_REPRO_TIMESTAMP` or the smoke default |
| Run type | `smoke` unless `CIVITAS_67B_REPRO_RUN_TYPE` explicitly changes it |
| Operator/environment notes | Research-only local sandbox run; no secrets or API keys required |

`RUN_ALL.sh` writes the run-specific manifest beside the run artifacts. That
manifest records the actual commit, dirty state, hashes, outputs, safety
counts, and command bindings for the run being reviewed.

- Model backend: `deterministic_baseline_stub`
- Model provider: `deterministic_stub`
- Backend kind: `stub`

  is absent
- Mode: local deterministic research stub
- Backend note: no live model backend, Cassius live invocation, Aegis promotion
  API, or Senate API is invoked by the smoke pipeline

Pilot and full manifests keep Gemma/Ollama as the primary reproducible lane:
provider `ollama`, backend kind `local`, run family `local_reproducible`,
default local tag `gemma4:e2b-it-q8_0`, and fallback tag `gemma3n:e2b` when
used. Any optional API/frontier lane must freeze provider, API backend kind,
model ID/version, API config status, settings, and whether that lane is
claim-supporting. Credentials never belong in a manifest; a skipped API lane
must retain `api_backend_enabled=false` and its skip reason.

## Safety Assertions

The verifier and run manifest assert:

- production mutation count must be zero
- unauthorized promotion count must be zero
- sandbox-only promotion count is recorded
- held-out and stress splits are evaluated only after sandbox promotion and are
  not candidate-generation inputs
- missing replay evidence is `not_available`, not passed replay evidence
- claim-supporting gate rows require passed, receipt-bound Cassius evidence;
  smoke uses Cassius `not_required` only by becoming non-claim-supporting

## Claim Boundary

This reproducibility bundle packages the current Civitas 6.7B research
pipeline. It does not add paper claims. A smoke run over placeholder tasks and
deterministic stub rows proves harness shape and auditability only. Sandbox
approval is not production approval.

## Limitations

- Default bundle runs are `smoke`, not pilot or full benchmark runs.
- Smoke baseline and held-out/stress behavior use deterministic stub adapters.
- Smoke train failures are exposed by a scored copy inside the run directory;
  source task and baseline artifacts are not mutated.
- Replay reproducibility is unavailable until replay rerun artifacts exist.
- Cassius is explicitly `not_required` in new smoke gate rows. Use
  `--cassius-challenge-evidence` for claim-supporting candidate gates.
- Live Aegis and Senate checks are not invoked by this smoke research adapter.
- Smoke outputs are not paper evidence.
